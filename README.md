# Интернет-магазин «Web-Larёk»
«Web-Larёk» — это интернет-магазин с товарами для веб-разработчиков, где пользователи могут просматривать товары, добавлять их в корзину и оформлять заказы. Сайт предоставляет удобный интерфейс с модальными окнами для просмотра деталей товаров, управления корзиной и выбора способа оплаты, обеспечивая полный цикл покупки с отправкой заказов на сервер.

## Архитектура приложения

Код приложения разделен на слои согласно парадигме MVP (Model-View-Presenter), которая обеспечивает четкое разделение ответственности между классами слоев Model и View. Каждый слой несет свой смысл и ответственность:

Model - слой данных, отвечает за хранение и изменение данных.  
View - слой представления, отвечает за отображение данных на странице.  
Presenter - презентер содержит основную логику приложения и  отвечает за связь представления и данных.

Взаимодействие между классами обеспечивается использованием событийно-ориентированного подхода. Модели и Представления генерируют события при изменении данных или взаимодействии пользователя с приложением, а Презентер обрабатывает эти события используя методы как Моделей, так и Представлений.

### Базовый код

#### Класс Component
Является базовым классом для всех компонентов интерфейса.
Класс является дженериком и принимает в переменной `T` тип данных, которые могут быть переданы в метод `render` для отображения.

Конструктор:  
`constructor(container: HTMLElement)` - принимает ссылку на DOM элемент, за отображение которого он отвечает.

Поля класса:  
`container: HTMLElement` - поле для хранения корневого DOM элемента компонента.

Методы класса:  
`render(data?: Partial<T>): HTMLElement` - Главный метод класса. Он принимает данные, которые необходимо отобразить в интерфейсе, записывает эти данные в поля класса и возвращает ссылку на DOM-элемент. Предполагается, что в классах, которые будут наследоваться от `Component` будут реализованы сеттеры для полей с данными, которые будут вызываться в момент вызова `render` и записывать данные в необходимые DOM элементы.  
`setImage(element: HTMLImageElement, src: string, alt?: string): void` - утилитарный метод для модификации DOM-элементов `<img>`


#### Класс Api
Содержит в себе базовую логику отправки запросов.

Конструктор:  
`constructor(baseUrl: string, options: RequestInit = {})` - В конструктор передается базовый адрес сервера и опциональный объект с заголовками запросов.

Поля класса:  
`baseUrl: string` - базовый адрес сервера  
`options: RequestInit` - объект с заголовками, которые будут использованы для запросов.  
  
Методы:  
`get(uri: string): Promise<object>` - выполняет GET запрос на переданный в параметрах ендпоинт и возвращает промис с объектом, которым ответил сервер  
`post(uri: string, data: object, method: ApiPostMethods = 'POST'): Promise<object>` - принимает объект с данными, которые будут переданы в JSON в теле запроса, и отправляет эти данные на ендпоинт переданный как параметр при вызове метода. По умолчанию выполняется `POST` запрос, но метод запроса может быть переопределен заданием третьего параметра при вызове.  
`handleResponse(response: Response): Promise<object>` - защищенный метод проверяющий ответ сервера на корректность и возвращающий объект с данными полученный от сервера или отклоненный промис, в случае некорректных данных.

#### Класс EventEmitter
Брокер событий реализует паттерн "Наблюдатель", позволяющий отправлять события и подписываться на события, происходящие в системе. Класс используется для связи слоя данных и представления.

Конструктор класса не принимает параметров.

Поля класса:  
`_events: Map<string | RegExp, Set<Function>>)` -  хранит коллекцию подписок на события. Ключи коллекции - названия событий или регулярное выражение, значения - коллекция функций обработчиков, которые будут вызваны при срабатывании события.

Методы класса:  
`on<T extends object>(event: EventName, callback: (data: T) => void): void` - подписка на событие, принимает название события и функцию обработчик.  
`emit<T extends object>(event: string, data?: T): void` - инициализация события. При вызове события в метод передается название события и объект с данными, который будет использован как аргумент для вызова обработчика.  
`trigger<T extends object>(event: string, context?: Partial<T>): (data: T) => void` - возвращает функцию, при вызове которой инициализируется требуемое в параметрах событие с передачей в него данных из второго параметра.


### Данные  

#### Интерфейс IProduct  
Содержит в себе поля для объявления единицы товара.  
Поля интерфейса:  
`id: string` - поле для хранения id товара.  
`title: string` - поле для хранения названия товара.  
`image: string` - поле для хранения пути у изображению товара.  
`category: string` - поле для хранения категории товара.  
`price: number | null` - поле для хранения цены на товар. Может принимать значение null, если товар бесценен.  
`description: string` - поле для хранения описания товара.  

#### Интерфейс IBuyer  
Содержит в себе поля данных покупателя.  
Поля интерфейса:  
`payment: TPayment` - поле для хранения метода оплаты. Может принимать только определенные значения.  
`address: string` - поле для хранения адреса получателя.  
`email: string` - поле для хранения электронной почты получателя.  
`phone: string` - поле для хранения номера телефона получателя.  

#### Интерфейс IApiGetResponseObject  
Содержит в себе поля получаемые при GET-запросе к серверу  
`total: number` - поле для хранения общего количества товаров  
`items: object[]` - поле для хранения массива объектов с данными по каждому товару.  

#### Интерфейс IApiPostRequestObject  
Наследует поля от интерфейса IApiGetResponseObject и IBuyer. Нужен для выполнения POST-запроса  

#### Интерфейс IApiPostResponseObject  
Содержит в себе поля получаемые при POST-запросе к серверу  
`id: string` - поле для хранения общего id заказа.  
`total: number` - поле для хранения общего количества товаров.  


### Модели данных  

#### Класс ProductCatalog  
Хранит товары, которые можно купить в приложении.  

Конструктор:  
`constructor()` - базовый конструктор без параметров.  

Поля класса:  
`__products: IProduct[]` - приватное поле для хранения массива всех товаров.  
`__currentProduct: IProduct` - приватное поле для хранения товара, выбранного для подробного отображения.  

Методы класса:  
`set products(array: IProduct[]): void` - метод для установки значения массиву товаров. В качестве параметра передается массив товаров.  
`get products(): IProduct[]` - метод для получения массива товаров. Возвращает массив товаров.  
`getProductById(id: string): IProduct` - метод для получения товара по его id. В качестве параметра в метод передается id товара. Метод возвращает объект типа IProduct.  
`set CurrentProduct(IProduct): void` - метод для сохранения товара, выбранного для подробного отображения. В качестве параметра товар IProduct.  
`get CurrentProduct(): IProduct | null`- метод для получения товара, выбранного для подробного отображения. Возвращает товар IProduct.  


#### Класс Cart  
Хранит товары, которые пользователь выбрал для покупки.  

Конструктор:  
`constructor()` - базовый конструктор без параметров.  

Поля класса:  
`__productsToBuy: IProduct[]` - поле для хранения массива всех товаров, добавленных покупателем в корзину.  

Методы класса:  
`get productsToBuy(): IProduct[]` - метод для получения массива товаров, которые находятся в корзине. Возвращает массив товаров, которые находятся в корзине.  
`addProductToBuy(product: IProduct): void` - метод для добавления товара в корзину. В качестве параметра передается товар IProduct, который добавляется в корзину.  
`deleteProductToBuy(product: IProduct): void` - метод для удаления товара из корзины. В качестве параметра передается товар IProduct, который удаляется из корзины.  
`clearCart(): void` - метод для очистки корзины.  
`countFullPrice(): Number`- метод для расчета цены всех товаров. Возвращает полную стоимость всех товаров, добавленных в корзину.  
`countCartItems(): Number` - метод для расчета количества товаров в корзине. Возвращает количество товаров в корзине.  
`isProductInCart(id: string): boolean` - метод для проверки наличия товара в корзине по его id. В качестве параметра передается id товара. Метод возвращает true, если товар с таким id найден в корзине, а false, если такого товара там нет.  

#### Класс Buyer  
Хранит данные покупателя, которые тот должен указать при оформлении заказа.  

Конструктор:  
`constructor()` - базовый конструктор без параметров.  

Поля класса:  
`__buyer` - поле для хранения всех данных покупателя.  
`__paymentType: TPayment` - поле для хранения вида оплаты, выбранного паользователем.  
`__address: string` - поле для хранения адреса получателя.  
`__email: string` - поле для хранения электронной почты получателя.  
`__phone: string` - поле для хранения номера телефона получателя.  
`__errors: { paymentType?: string; address?: string; email?: string; phone?: string }` - поле для хранения ошибок в виде объекта. Если поле невалидно, то в объект добавляется соответствующая пара ключ-значение.  

Методы класса:  
`get errors(): { paymentType?: string; address?: string; email?: string; phone?: string }` - метод для получение объекта с ошибками. Возвращает объект с ошибками.  
`set paymentType(paymentType: TPayment): void` - метод для заполнения поля paymentType. В качестве параметра принимает paymentType типа TPayment.  
`isPaymentTypeValid(): boolean` - метод проверяющий валидность поля paymentType. Возвращает true, если оно валидно, иначе false.  
`set address(address: string): void` - метод для заполнения поля address. В качестве параметра принимает address типа string.  
`isAddressValid(): boolean` - метод проверяющий валидность поля address. Возвращает true, если оно валидно, иначе false.  
`set email(email: string): void` - метод для заполнения поля email. В качестве параметра принимает email типа string.  
`isEmailValid(): boolean` - метод проверяющий валидность поля email. Возвращает true, если оно валидно, иначе false.  
`set phone(phone: string): void` - метод для заполнения поля phone. В качестве параметра принимает phone типа string.  
`isPhoneValid(): boolean` - метод проверяющий валидность поля phone. Возвращает true, если оно валидно, иначе false.  
`getAllUserData(): buyer: IBuyer`- метод для получения всех данных покупателя. Возвращает покупателя IBuyer.  
`clearAllUserData(): void`- метод для удаления всех данных покупателя.  


### Слой коммуникации  

#### Класс CommunicationService  
Отвечает за получение данных с сервера и отправку данных на сервер.  

Конструктор:  
`constructor(apiObj: IApi)` - принимает объект, соответствующий интерфейсу IApi для дальнейшей работы с сервером.  

Поля класса:  
`api` - поле для хранения объекта, при помощи которого будут выполняться запросы к серверу  

Методы класса:  
`getProducts()` - асинхронный метод, выполняющий GET-запрос к серверу. Получает от сервера массив products и общее количество товаров. Возвращает Promise, который при успешном выполнении разрешается в массив products.  
`makeOrder(data: IApiRequestObject)` - метод, выполняющий POST-запрос к серверу. В качестве параметра получает объект, который будет передан серверу в качестве body.

### Слой представления  

#### Класс Card
Служит базовым классом для карточек товаров, отвечающим за отображение названия и цены. Наследуется от класса Component и используется как родитель для карточек в каталоге, корзине и модальном окне.

Конструктор:  
`constructor(container: HTMLElement, events?: IEvents)` — принимает HTML-контейнер карточки и объект событий. Инициализирует DOM-элементы названия и цены товара.

Поля класса:  
`_cardTitle: HTMLElement` — DOM-элемент, отвечающий за отображение названия товара.  
`_cardPrice: HTMLElement` — DOM-элемент, отвечающий за отображение цены товара.  
`events?: IEvents` — объект для взаимодействия с брокером событий (необязательный).

Методы класса:  
`set title(value: string)` — устанавливает текст заголовка карточки.  
`set price(value: number | null)` — устанавливает цену товара. Если значение `null`, отображает текст «Бесценно».

#### Класс CardFromCart
Служит для взаимодействия с карточкой товара внутри корзины. Наследуется от класса Card и расширяет его функциональностью удаления товара и отображения его позиции в списке.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — принимает HTML-контейнер карточки и объект действий. Инициализирует кнопку удаления и элемент индекса товара. Если передан обработчик `onClick`, навешивает его на кнопку удаления.

Поля класса:  
`_deleteButton: HTMLButtonElement` — кнопка удаления товара из корзины.  
`_itemIndex: HTMLElement` — элемент, отображающий порядковый номер товара в корзине.  

Методы класса:  
`set itemIndex(value: number)` — устанавливает индекс позиции товара в корзине.

#### Класс CardFromCatalog
Служит для взаимодействия с карточкой товара в каталоге. Наследуется от класса Card и дополняет его возможностью отображения изображения и категории товара, а также обработкой клика по всей карточке.

Конструктор:  
`constructor(container: HTMLElement, actions?: ICardActions)` — принимает HTML-контейнер карточки и объект действий. Инициализирует DOM-элементы изображения и категории. Если передан обработчик `onClick`, назначает его на контейнер карточки.

Поля класса:  
`_cardImage: HTMLImageElement` — DOM-элемент изображения товара.  
`_cardCategory: HTMLElement` — DOM-элемент категории товара.  

Методы класса:  
`set title(value: string)` — устанавливает название товара и обновляет атрибут `alt` изображения.  
`set category(value: string)` — устанавливает категорию товара и обновляет CSS-классы в соответствии с `categoryMap`.  
`set image(value: string)` — устанавливает изображение товара, автоматически заменяя `svg` → `png` и формируя полный путь с использованием `CDN_URL`.  

#### Класс CardFullDescribtion
Служит для отображения товара в полном описании внутри модального окна. Наследуется от класса Card и расширяет его функциональностью отображения изображения, категории, описания и кнопки покупки. Поддерживает изменение состояния кнопки в зависимости от того, находится ли товар в корзине.

Конструктор:  
`constructor(container: HTMLElement, events: IEvents, actions?: ICardActions)` — принимает DOM-контейнер карточки, объект событий и набор действий. Инициализирует DOM-элементы изображения, категории, описания и кнопки покупки. Если передан обработчик `onClick`, назначает его на кнопку.

Поля класса:  
`_cardImage: HTMLImageElement` — изображение товара.  
`_cardCategory: HTMLElement` — категория товара.  
`_cardDescription: HTMLElement` — описание товара.  
`_buyButton: HTMLButtonElement` — кнопка добавления или удаления товара из корзины.  
`id: string` — идентификатор товара.

Методы класса:  
`set title(value: string)` — устанавливает название товара и обновляет `alt` изображения.  
`set category(value: string)` — устанавливает категорию товара и обновляет CSS-классы категории согласно `categoryMap`.  
`set image(value: string)` — устанавливает изображение товара с автоматической заменой `svg` → `png` и использованием `CDN_URL`.  
`set description(value: string)` — устанавливает текст описания товара.  
`set buyButton(isInCart: boolean)` — изменяет текст кнопки покупки в зависимости от статуса товара в корзине. Если товар имеет цену `Бесценно`, кнопка отключается и отображает «Недоступно».

#### Класс CartView
Служит для отображения содержимого корзины в модальном окне, включая список товаров, общую стоимость и состояние кнопки оформления заказа. Наследуется от класса Component.

Конструктор:  
`constructor(events: IEvents, container: HTMLElement)` — принимает объект событий и контейнер корзины. Инициализирует список товаров, кнопку оформления и элемент отображения цены. Назначает обработчик клика по кнопке оформления, вызывающий событие `purchase:open`.

Поля класса:  
`_productsList: HTMLElement` — контейнер для списка товаров, добавленных в корзину.  
`_buyButton: HTMLButtonElement` — кнопка перехода к оформлению заказа.  
`_priceElement: HTMLElement` — элемент, отображающий текущую сумму заказа.  
`events: IEvents` — объект для работы с системой событий.

Методы класса:  
`set productList(elements: HTMLElement[])` — добавляет переданные элементы товаров в список корзины.  
`set buttonState(isValid: boolean)` — включает или выключает кнопку оформления заказа в зависимости от переданного значения.  
`set price(currentPrice: number)` — обновляет отображаемую общую стоимость товаров в корзине.

#### Класс Contacts
Служит для управления формой ввода контактных данных покупателя — электронной почты и номера телефона. Наследуется от класса Form и обеспечивает связь с системой событий при вводе данных.

Конструктор:  
`constructor(events: IEvents, container: HTMLFormElement)` — принимает объект событий и HTML-элемент формы. Инициализирует поля ввода email и телефона, а также навешивает обработчик ввода, который отправляет события изменения данных.

Поля класса:  
`_email: HTMLInputElement` — поле ввода электронной почты.  
`_phone: HTMLInputElement` — поле ввода номера телефона.  
`events: IEvents` — объект для отправки событий об изменении данных.

Методы класса:  
`set email(value: string)` — устанавливает значение поля ввода email.  
`set phone(value: string)` — устанавливает значение поля ввода телефона.

#### Класс Form
Служит базовым классом для работы с формами, объединяя логику валидации, отображения ошибок и управления состоянием кнопки отправки. Наследуется от класса Component.

Конструктор:  
`constructor(container: HTMLFormElement, events: IEvents)` — принимает HTML-форму и объект событий. Инициализирует кнопку отправки и контейнер ошибок. Назначает обработчик события `submit`, предотвращает стандартное поведение и отправляет событие `<имя_формы>:submit`.

Поля класса:  
`_submitButton: HTMLButtonElement` — кнопка отправки формы.  
`_errors: HTMLElement` — контейнер для отображения ошибок.  
`events: IEvents` — объект, используемый для отправки событий формы.

Методы класса:  
`set errors(value: string)` — устанавливает текст ошибки в контейнер ошибок.  
`set buttonState(isValid: boolean)` — включает или выключает кнопку отправки формы в зависимости от валидности данных.

#### Класс Gallery
Служит для отображения списка карточек товаров на главной странице. Наследуется от класса Component и управляет наполнением галереи.

Конструктор:  
`constructor(events: IEvents, container: HTMLElement)` — принимает объект событий и DOM-контейнер галереи. Передаёт контейнер в родительский класс.

Поля класса:  
`_galleryItems: HTMLElement[]` — массив DOM-элементов карточек товаров, отображаемых в галерее.  

Методы класса:  
`set galleryItems(items: HTMLElement[])` — добавляет переданные элементы карточек в контейнер галереи в порядке их следования.

#### Класс Header
Служит для управления элементами шапки сайта, включая кнопку корзины и счётчик добавленных товаров. Наследуется от класса Component.

Конструктор:  
`constructor(events: IEvents, container: HTMLElement)` — принимает объект событий и контейнер шапки. Инициализирует кнопку корзины и счётчик товаров, а также вешает обработчик клика по кнопке, вызывающий событие `basket:open`.

Поля класса:  
`_headerBasket: HTMLButtonElement` — кнопка открытия корзины.  
`_basketCounter: HTMLElement` — элемент отображения количества товаров в корзине.  
`events: IEvents` — объект событий.

Методы класса:  
`set basketCounter(value: number)` — обновляет отображаемый счётчик товаров в корзине.

#### Класс Modal
Служит для управления модальным окном, включая его открытие, закрытие и замену содержимого. Наследуется от класса Component.

Конструктор:  
`constructor(events: IEvents, container: HTMLElement)` — принимает объект событий и контейнер модального окна. Инициализирует контейнер содержимого и кнопку закрытия. Назначает обработчик клика, вызывающий событие `modal:close`.

Поля класса:  
`_modalContent: HTMLElement` — контейнер для отображения содержимого модального окна.  
`_modalCloseButton: HTMLButtonElement` — кнопка закрытия модального окна.  
`events: IEvents` — объект событий.

Методы класса:  
`set modalContent(content: HTMLElement)` — очищает текущее содержимое модального окна и вставляет новое.  
`open()` — активирует модальное окно, добавляя класс `modal_active`.  
`close()` — скрывает модальное окно, удаляя класс `modal_active`.

#### Класс Order
Служит для управления формой оформления заказа, включая выбор способа оплаты и ввод адреса доставки. Наследуется от класса Form.

Конструктор:  
`constructor(container: HTMLFormElement, events: IEvents)` — принимает контейнер формы и объект событий. Инициализирует кнопки выбора метода оплаты и поле ввода адреса. Назначает обработчики кликов по кнопкам оплаты и обработчик ввода адреса, которые отправляют соответствующие события.

Поля класса:  
`_paymentType: HTMLButtonElement[]` — список кнопок выбора способа оплаты.  
`_address: HTMLInputElement` — поле ввода адреса доставки.  
`events: IEvents` — объект событий.

Методы класса:  
`set paymentType(method: string)` — визуально переключает активный способ оплаты: добавляет или убирает класс `button_alt-active` и обновляет состояние `ariaPressed`.  
`set address(value: string)` — устанавливает значение поля ввода адреса.

#### Класс Success
Служит для отображения окна успешного оформления заказа, включая итоговую стоимость и кнопку возврата. Наследуется от класса Component.

Конструктор:  
`constructor(events: IEvents, container: HTMLElement)` — принимает объект событий и контейнер модального окна. Инициализирует элементы описания успешного заказа и кнопку завершения. На кнопку навешивается обработчик, отправляющий событие `success:order`.

Поля класса:  
`_orderSuccessDescription: HTMLElement` — элемент, отображающий итоговую информацию о списанной сумме.  
`_orderSuccessButton: HTMLButtonElement` — кнопка завершения и возврата к каталогу.  
`events: IEvents` — объект событий.

Методы класса:  
`set totalPrice(value: number)` — обновляет текст описания, показывая списанную сумму в синапсах.
